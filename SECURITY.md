# Security Policy

The Ireland Government Design System takes security seriously. We appreciate the efforts of security researchers and the broader community in helping us maintain the safety and integrity of government services.

## Reporting a vulnerability

**Please do not open public GitHub issues, pull requests, or discussions for security concerns.**

Instead, report vulnerabilities through one of the following channels:

- **GitHub:** [Report a vulnerability](https://github.com/ogcio/govie-ds/security/advisories/new)
- **Email:** [appsarchitecture@dsu.cloud.gov.ie](mailto:appsarchitecture@dsu.cloud.gov.ie)

### What to include

- A description of the vulnerability
- Steps to reproduce the issue
- Affected version(s) and environment details
- Potential impact or severity assessment
- Proof of concept, if available

## Responsible disclosure

We ask that security researchers:

- Allow us reasonable time to investigate and address the issue before public disclosure
- Avoid accessing or modifying data that does not belong to you
- Do not perform actions that could negatively impact other users (e.g., denial of service)
- Make a good faith effort to avoid privacy violations

## Scope

This policy applies to:

- Code in this repository
- Published packages from this repository
- Documentation that could lead to misuse
- Patterns that may introduce security or fraud risks if misunderstood

## What this design system does not do

This design system:

- Does not provide authentication or identity guarantees
- Does not validate official government services
- Should not be used as proof of legitimacy by third parties

Security concerns related to impersonation or fraudulent use of government branding will be treated seriously and escalated appropriately.

Thank you for helping keep government services safe.
