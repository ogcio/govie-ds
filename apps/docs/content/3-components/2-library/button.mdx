## Usage

<ComponentPropertiesTable componentId="button" />

### Button Props

- **children** (optional): The content inside the button. Can be a string or any valid React node.

- **variant** (optional): Defines the visual style of the button. Options include:

  - `'primary'`
  - `'secondary'`
  - `'flat'`

- **appearance** (optional): Adjusts the overall tone or theme of the button. Options are:

  - `'default'`
  - `'dark'`
  - `'light'`

- **size** (optional): Specifies the size of the button. Options include:

  - `'small'`
  - `'medium'`
  - `'large'`

- **disabled** (optional): A boolean that, when `true`, disables the button and prevents interaction.

### ButtonGroup Props

Used to display a collection of related buttons (e.g., for feedback, selection, etc.).

- **name** (required): A string used to uniquely identify the button group. Useful for forms or accessibility purposes.

- **size** (optional): Defines the size of each button within the group. Options include:

  - `'small'`
  - `'medium'`
  - `'large'`

- **onChange** (optional): A callback function triggered when a button is selected. It receives the selected value.

- **defaultValue** (optional): The initially selected value in the group.

- **children** (optional): Can be used to inject custom React nodes into the group (if needed for flexibility).

### Best Practices

#### When to use this component

- Use **Button** for user-triggered actions like submitting a form or confirming a choice.
- Use **ButtonGroup** when offering a set of mutually exclusive options, such as rating selections or binary questions.

#### When not to use this component

- Don’t use **Button** when the goal is navigation — use a **Link** component instead.
- Don’t use **ButtonGroup** if you only need a single action — use just a **Button** in that case.
