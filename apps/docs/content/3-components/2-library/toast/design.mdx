# Toast

Toasts are brief, non-blocking notifications that inform users about the outcome of an action or system event. They appear temporarily and auto-dismiss after a short duration, without requiring user focus.

# Variants

Use variants to communicate intent:

- **Info** — neutral updates.
- **Success** — positive confirmations.
- **Warning** — cautionary messages.
- **Danger** — errors and blocking issues.

# Placement & Position

Toasts should not obscure critical UI. Preferred defaults:

- Desktop: bottom-right
- Tablet/Mobile: bottom (full width when space is tight)

### Best Practices

- Use dismissible toasts when the message is not critical.
- Keep animations smooth and consistent with UI interactions.
- Positioning matters: Important alerts should appear in prominent areas like top-right or center.

## Do

- System notifications (e.g., "Form submitted successfully.")
- Warnings & Errors that require user attention.
- Temporary alerts that disappear automatically.

  <ComponentContainer>
    <Toast
      title="Saved"
      description="Your changes were saved."
      variant="success"
    />
  </ComponentContainer>

## Don't

- Permanent messages → Use modals or banners instead.
- Multiple simultaneous alerts → Can overwhelm users.
