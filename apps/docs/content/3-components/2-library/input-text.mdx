# Input Text

The `InputText` component is used to allow users to enter single-line text, with options to adjust the width dynamically using `halfFluid`, `fullFluid`, or `characterWidth`. It can be used with prefixes, suffixes, and error states for flexible form input scenarios.

## Usage

<ComponentPropertiesTable componentId="input-text" />

#### Features:

- Includes a built-in reset button to clear the input field.
- Maintains the same styling and customisation options as the standard input component.
- Ideal for search fields, dynamic form inputs, or scenarios requiring frequent text updates.

### Text Input Props

- **prefix** (optional): Element or text to display on the left side of the input, such as a unit or symbol.
- **suffix** (optional): Element or text to display on the right side of the input, such as a unit or symbol.
- **halfFluid** (optional): If `true`, the text input will take up half the width of the container.
- **ref** (optional): A reference to the input element.
- **type** (optional): Specifies the input type.
- **disabled** (optional): Disable the input.
- **clearEnabled** (optional): Enhances user experience by displaying a reset (clear) button inside the input field. When enabled, users can quickly clear the input value with a single click. This is particularly useful in forms where users frequently modify or re-enter information. It ensures fast interaction without requiring manual deletion.
- **Other InputHTMLAttributes** (optional): The component also accepts other standard HTML input attributes, such as `placeholder`, `maxLength`, etc., through the `InputHTMLAttributes` generic props.

### Best Practices

### When to Use This Component

- **Single-Line Text Inputs**: Use this component for short, single-line inputs such as names, email addresses, phone numbers, and short codes (e.g., postal codes).

- **Customizable Widths**: Utilise the `halfFluid` or `fullFluid` properties for inputs that need to fit in responsive layouts. The `characterWidth` prop allows you to specify a custom width based on the number of characters the input should accommodate.

- **Error States**: For forms where invalid input needs to be flagged, the `hasError` prop visually indicates errors with a red border.

- **Custom Prefixes or Suffixes**: This component supports adding contextual information next to the input (e.g., units, symbols) using `prefix` and `suffix` props.

### When Not to Use This Component

- **Multi-Line Text Fields**: If your input requires multiple lines (such as a message box), use a `textarea` instead of `InputText`.
- **Input with Formatting Requirements**: If input masking or format validation (like date pickers or rich text editors) is necessary, consider using a specialised component.

- **Non-Textual Inputs**: This component is not suitable for non-textual input types like checkboxes, radio buttons, or dropdown menus.
