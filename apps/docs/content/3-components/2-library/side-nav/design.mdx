# Side Navigation

Side Navigation (SideNav) is used to organize and access major sections of an application. It provides persistent and hierarchical navigation, making it ideal for dashboards or multi-section layouts.

## Variants

The SideNav component supports a variety of features, including primary items for top-level navigation and expandable sections that reveal nested secondary links.
It also offers icon support through Material Symbols to provide visual context and allows for controlled open and closed states for expandable sections.

## Usage

The Side Navigation component is ideal for application layouts that require persistent, structured vertical navigation across different views.
It is particularly useful in scenarios where the interface includes multiple sections or tools, when there is a need to organize pages hierarchically, and when keeping navigation always visible enhances quick access and switching between views.

### Do

- Use for persistent navigation in multi-section layouts.
- Provide clear and distinct labels for items.
- Use `expandable` for parent items with nested links.
- Use icons to reinforce meaning for top-level items.

<ComponentContainer>
  <SideNav value="team-permissions">
    <SideNavItem primary expandable value="team" label="Team">
      <SideNavItem secondary value="team-members" label="Members" />
      <SideNavItem secondary value="team-permissions" label="Permissions" />
    </SideNavItem>
  </SideNav>
</ComponentContainer>

### Don't

- Don’t use it for simple, flat navigation with just a few items.
- Don’t mix unrelated links in the same group.
- Don’t overload with deep hierarchies, keep nesting shallow and clear.

<ComponentContainer>
  <SideNav value="settings">
    <SideNavItem primary value="settings" label="Settings" />
    <SideNavItem primary value="preferences" label="Preferences" />
    <SideNavItem
      primary
      value="terms"
      label="Terms and Conditions and Legal Disclaimers"
    />
  </SideNav>
</ComponentContainer>
