# Input Password

The Input Password component allows users to securely enter and toggle the visibility of password fields. It includes built-in visibility controls and can be used anywhere password input is required.

## Usage

Use the Input Password component for fields requiring secure input, such as account creation, login forms, or security settings.

### Do

- **Allow users to toggle visibility**: Let users confirm their input when necessary.
- **Use clear hints and error messages**: Guide users on password requirements and show helpful feedback.
- **Ensure consistent styling**: Match the component to your overall design system for clarity and trust.

**Example:**

<ComponentContainer>
  <div className="flex flex-col gap-4">
    <FormField>
      <FormFieldLabel htmlFor="text-password-id">Password</FormFieldLabel>
      <InputPassword placeholder="Placeholder" data-testid="text-password-id" />
    </FormField>
    <div className="gi-text-sm gi-text-gray-600">
      Includes visibility toggle and secure input
    </div>
  </div>
</ComponentContainer>

### Don't

- **Don’t remove the visibility toggle**: Users often need to verify their input, especially on mobile.
- **Don’t neglect validation**: Always validate password inputs and inform the user when rules are not met.
- **Don’t expose password rules too late**: Show guidance before submission to reduce friction.

**Example:**

<ComponentContainer>
  <div className="flex flex-col gap-4">
    <InputPassword placeholder="No guidance, hidden errors" />
    <div className="gi-text-sm gi-text-gray-600">
      Missing hints or rules – user may get frustrated
    </div>
  </div>
</ComponentContainer>
