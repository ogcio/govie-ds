## Usage

Input fields are used to capture user data like text, email, numbers, and more.

### Do

- **Use labels**: Always pair input fields with a clear and accessible label.
- **Use appropriate types**: Choose input types like email, password, or number to improve user experience.
- **Provide placeholders**: Use descriptive placeholder text as a hint for what the input expects.
- **Support clearing input**: When applicable, allow users to reset input with a clear button.
- **Include icons when helpful**: Use iconStart or iconEnd for visual hints (e.g., search, calendar).
- **Use prefix/suffix for formatting**: Display currency, units, etc., clearly.
- **Use a secondaryLabel property for secondary inline context**: Add brief, non-essential hints like “(optional)” or “(beta)” with `secondaryLabel`. Keep it short.

### Don't

- **Don’t rely only on placeholder text**: Always use labels to avoid confusion.
- **Don’t overload inputs with unrelated elements**: Prefix, suffix, and icons should be relevant to the input’s context.
- **Don’t use vague placeholders**: Be specific about the expected value.
