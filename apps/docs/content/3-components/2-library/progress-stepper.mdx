## Usage

### ProgressStepper Props

- **children** (required): An array of `StepItem` components, each representing an individual step.
- **currentStepIndex** (optional): A number (zero-based index) representing the current active step. Defaults to `0`.
- **orientation** (optional): A string defining the layout of the progress stepper, either `horizontal` or `vertical`. The default is `horizontal`.
- **completeAll** (optional): A boolean that, when set to `true`, marks all steps as completed, regardless of the current step. This can be useful in scenarios like post-submission actions.
- **indicator** (optional): Defines how each step indicator is rendered. Accepts `number` or `hashtag`. When set to `number`, each step is shown as its index. When `hashtag`, all steps display the # symbol instead. Defaults to `hashtag`.

### StepItem Props

- **label** (required): A string representing the label for the step, providing context for the user on what this step is about.
- **children** (optional): The content of the step, which can include any valid React node (e.g., forms, string, etc.).
- **defaultOpen** (optional): A boolean indicating whether the step should be open by default. When set to `true`, the step is visible when the component first renders.

### Best Practices

- Use Progress Stepper to guide users through a linear workflow or process.
- Display clear labels for each step to provide context and direction.

## When to use this component

- Use Progress Stepper when guiding users through a multi-step process (e.g., filling out forms, completing onboarding steps, or navigating a wizard).
- Display each step in a clear and concise manner using the `StepItem` component to make each stage in the process explicit.

## When not to use this component

- Avoid using Progress Stepper when the steps do not follow a sequential flow or are non-linear.
- Do not use Progress Stepper for workflows that allow users to jump between steps without any defined order or progression.
