# Header

The Header component functions as a top-level navigation element for digital services across government departments and public sector organisations.
It offers users a consistent and recognisable interface that builds trust, supports a unified brand identity, and delivers key navigation tools.
Originally referred to as the GOV.IE masthead, this component is designed to signal to users that they are engaging with an official government service while providing easy access to essential site-wide features. It is adaptable for use beyond GOV.IE, supporting a broad range of departmental and agency websites.

## Structure

**Composable by design.** The Header now uses explicit primitives instead of the old single `<Header items secondaryLinks />` props API. Compose the layout with:

- `HeaderLogo`, `HeaderTitle`
- `HeaderPrimaryMenu`, `HeaderSecondaryMenu`
- `HeaderMenuItemLink`, `HeaderMenuItemButton`, `HeaderMenuItemSeparator`
- `HeaderSlotContainer` (for dropdown/inline regions), `DrawerWrapper`/`DrawerBody`/`DrawerFooter` (for dialogs/drawers)

**Why this is better (vs. legacy `items/secondaryLinks`):**

- Precise semantics & a11y: direct control over roles, labels, `aria-*`, and focus order.
- Testability: deterministic targets (IDs, roles) for testing.
- Layout flexibility: exact ordering and responsive visibility through `showItemMode`.

**Quick migration map:**

- `items[].itemType === 'link'` ⟶ `<HeaderMenuItemLink {...} />`
- `items[].itemType === 'divider'` ⟶ `<HeaderMenuItemSeparator />`
- `items[].itemType === 'slot'` (e.g. search) ⟶ `<HeaderMenuItemButton ... aria-controls="X" />` + `<HeaderSlotContainer id="X">…</HeaderSlotContainer>`
- `secondaryLinks[]` ⟶ `<HeaderSecondaryMenu>` with `<HeaderMenuItemLink>` children

## Usage

The GOV.IE header should appear at the top of every page on your service. Use it to clearly identify the service (title or logo), provide meaningful top-level navigation, and expose useful site-wide tools (search, language, etc.).

### Do

- Place the header at the top of every page.
- Use a concise service **title** or an approved **logo**.
- Link the logo to the home page.
- Keep the **primary menu** short and task-focused.
- Use `HeaderMenuItemSeparator` to group related links when needed.
- Expose global tools via a button + `HeaderSlotContainer` (dropdown/inline) or `DrawerWrapper` where appropriate.
- Use `showItemMode="desktop-only" | "mobile-only"` for visibility, not CSS hacks.
- Provide a **mobile menu** trigger when primary items won't fit.
- Set `aria-label` on the header and navigation regions.
- Ensure every control has an accessible name; keep icon text present inside the button.

### Don't

- Don't use long titles; they break layout and readability.
- Don't overload the primary menu with too many items.
- Don't stack multiple headers on one page.
- Don't place complex, multi-step flows inside the header area.
