# Header

The Header component serves as the top-level navigation element across all GOV.IE digital services. 
It provides users with a consistent and recognisable interface that establishes trust, reinforces brand identity, and offers essential navigation tools. 
This component, sometimes referred to as the GOV.IE masthead, helps users instantly identify they're interacting with an official government service while providing access to site-wide functionality.

### Header with Custom Links

You can also use custom links components with the header component.
For instance, you can use the `react-router-link` or `next/link` component to link to a specific route in your application.

## Usage

<ComponentPropertiesTable componentId="header" />

### `HeaderProps`

Props for the `Header` component:

- **`title`** (optional): The title of the header.
- **`logo`** (optional): An object defining logo properties:
  - `imageSmall`: Small version of the logo.
  - `imageLarge`: Large version of the logo.
  - `href`: URL to navigate to when the logo is clicked.
  - `external`: If true, opens the logo link in a new tab.
  - `alt`: Alternative text for the logo image.
- **`addDefaultMobileMenu`** (optional): If true, the default mobile menu is included in the header.
- **`mobileMenuLabel`** (optional): The label for the mobile menu.
- **`items`** (optional): A list of `HeaderItem` objects to include in the header.
- **`secondaryLinks`** (optional): A list of secondary links for the header.
  - `href`: The URL for the link.
  - `label`: The label for the link.
  - `slot`: A ReactNode element. It's recommended to wrap it with a `Paragraph` component or a link component (e.g., NextLink).
- **`fullWidth`** (optional): If true, the header will span the full width of the container.
- **`showTitleOnMobile`** (optional): If true, the header title will be shown on mobile.
- **`dataTestid`** (optional): Custom test ID for the header.

### `HeaderItemMode`

- **`'always'`**: The item is always displayed.
- **`'mobile-only'`**: The item is displayed only on mobile devices.
- **`'desktop-only'`**: The item is displayed only on desktop devices.

### `HeaderItemAppearance`

- **`'dropdown'`**: The item is displayed as a dropdown menu.
- **`'drawer'`**: The item is displayed within a drawer.

### `HeaderItemType`

- **`'slot'`**: A custom slot where you can place custom content. This slot can have additional properties:
  - `label`: An optional label for the slot.
  - `icon`: An optional icon to display alongside the content.
  - `component`: A required React component to render inside the slot.
  - `slotAppearance`: The appearance of the slot, which can either be a dropdown or drawer.
  - `drawerPosition`: Specifies the position of the drawer if it's being used.
- **`'divider'`**: A visual divider, used to separate other header items. This type does not require any additional props except for the `itemType` property.

- **`'link'`**: A clickable link. It can have the following properties:

  - `label`: The label to display for the link.
  - `icon`: An optional icon to display alongside the label.
  - `href`: The URL to which the link should point.
  - `external`: If true, opens the link in a new tab.
  - `onClick`: An optional click handler for the link.

- **`'custom-link'`**: A custom link, rendered with a React component. This type only requires the `component` property.

### `HeaderItem`

A type that combines both `CommonProps` and `ConditionalProps`, representing an individual item in the header.

- **`CommonProps`**:

  - `showItemMode` (optional): Specifies when the header item should be displayed, based on the `HeaderItemMode` type.

- **`ConditionalProps`**:
  These props vary based on the `itemType` of the header item. Here’s the breakdown for each `itemType`:

  - **`'slot'`**:
    - `label`: Optional label for the slot.
    - `icon`: Optional icon.
    - `component`: The React component to display.
    - `slotAppearance`: Optional. Defines if the slot should have a dropdown or drawer appearance.
    - `drawerPosition`: Optional. Defines the drawer position when using a drawer appearance.
  - **`'divider'`**:

    - This item type doesn't require any additional properties besides `itemType`.

  - **`'link'`**:
    - `label`: The text for the link.
    - `icon`: An optional icon.
    - `href`: The URL the link will go to.
    - `external`: If true, opens the link in a new tab.
    - `onClick`: Optional click handler for the link.
  - **`'custom-link'`**:
    - `component`: The custom React component that renders the link content.

### Best Practices

### When to use this component

You must use the GOV.IE header component at the top of every page. The Service Manual explains why it’s important for you to make your service look like GOV.IE.

Additionally, the header now includes a "slots" feature, allowing you to render specific components below the header. This is useful for adding custom elements like navigation aids, search boxes, or additional contextual information that complements the header's main functionality.

### When not to use this component

If your service is not hosted on one of the GOV.IE domains outlined, you must not use the GOV.IE header component as it’s not considered part of GOV.IE.
You can still build from this component, but you’ll need to make some changes to make sure users do not confuse your website with GOV.IE.

The slots feature lets you add unique, context-specific content directly below the header to support user navigation and service-specific functionality. Examples include displaying language selectors, search input fields, or quick links for frequently accessed features.

### How it works

The GOV.IE header component ensure users get a consistent experience on GOV.IE.

This also assures users that they’re in the right place to use your service and to understand that GOV.IE functions as one website.

For guidance on how to plan your header and navigation, see the Help users navigate a service pattern.

The slots can be used to augment this pattern by including additional, user-friendly navigation to your service's needs.
