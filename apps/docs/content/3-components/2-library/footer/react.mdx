## Migration guide

The slot-based Footer API from `@ogcio/design-system-react` is **deprecated**. Migrate to the composable primitives from `@ogcio/design-system-react/next`:

```jsx
// Before
import { Footer } from '@ogcio/design-system-react';

// After
import {
  Footer,
  FooterSection,
  FooterLogo,
} from '@ogcio/design-system-react/next';
```

**Why migrate:**

- Layout flexibility: you own dividers, container padding, grids, and logo placement.
- Precise semantics and a11y: set `ariaLabel` and region labels yourself.
- Clearer JSX: named children (`FooterSection`, `FooterLogo`) instead of opaque slot props.

### Before and after

**Before** - legacy slots:

```jsx
import { Footer } from '@ogcio/design-system-react';

<Footer
  primarySlot={<div>{/* primary links */}</div>}
  secondarySlot={<div>{/* secondary links */}</div>}
  utilitySlot={<div>{/* utility links */}</div>}
  logo={{
    href: '/',
    alt: 'Gov.ie Logo',
  }}
  dataTestid="footer"
/>;
```

The legacy component wrapped those slots for you (container, divider when both primary and secondary were set, logo positioning, and landmark `aria-label`).

**After** - compose the equivalent structure:

```jsx
import {
  Footer,
  FooterSection,
  FooterLogo,
} from '@ogcio/design-system-react/next';
import { LogoGoldGreen } from '@ogcio/design-system-react/logos';
import { Container, Divider, Stack, Box } from '@ogcio/design-system-react';

<Footer ariaLabel="Footer" dataTestId="footer">
  <FooterSection>
    <Container>
      {/* former primarySlot */}
      <Box>{/* primary links */}</Box>

      <Divider className="gi-my-8" />

      <Stack direction={{ base: 'column', md: 'row' }} gap={{ base: 0, md: 6 }}>
        {/* former secondarySlot */}
        <Box>{/* secondary links */}</Box>
        <Link href="/" ariaLabel="Homepage">
          <FooterLogo>
            <LogoGoldGreen size="181" label="Gov.ie Logo" />
          </FooterLogo>
        </Link>
      </Stack>
    </Container>
  </FooterSection>

  <FooterSection variant="utility">
    {/* former utilitySlot */}
    <Box>{/* utility links */}</Box>
  </FooterSection>
</Footer>;
```

Key differences when transforming:

1. **One primary section** - `primarySlot` and `secondarySlot` share a single `<FooterSection>`. Separate them with a `<Divider>` only when you had both slots.
2. **Logo is a sibling** - place `<FooterLogo>` next to secondary content inside a `Stack` with `direction={{ base: 'column', md: 'row' }}`, not nested under the secondary links.
3. **Utility stays separate** - move `utilitySlot` into `<FooterSection variant="utility">`.
4. **Accessibility is explicit** - set `ariaLabel` on `<Footer>`. Legacy applied this automatically.
5. **Logo placement** - For mobile screens, the logo is now positioned below any preceding content by default.

### Migrating the `logo` prop

Legacy accepted `href`, `alt`, `imageLarge`, `imageSmall`, `width`, `height`, and `external`, and rendered a linked logo in the secondary area when `href` was set.

In the composable API, `FooterLogo` only handles positioning. Render the image yourself:

```jsx
// Before
<Footer
  logo={{
    href: '/',
    alt: 'Gov.ie Logo',
    imageLarge: '/logo-desktop.svg',
    imageSmall: '/logo-mobile.svg',
    external: false,
  }}
  secondarySlot={…}
/>

// After
<FooterLogo>
  <LogoGoldGreen size="181" label="Gov.ie Logo" />
</FooterLogo>
```

- For custom assets, use your own `<img>` or `<picture>` with `alt` text.
- To keep a home link (optional), wrap the logo in `<Link>` yourself and set an accessible name, for example `ariaLabel="Go to the home page"`. Use `external` on `<Link>` if the destination is off-site.
- `FooterLogo` does not add a link, alt text, or image for you.

### Related layout pieces

- Use [`Container`](/components/library/container/design/) for the horizontal padding the legacy footer applied by default.
- Use a `Stack` with `direction={{ base: 'column', md: 'row' }}` for secondary links beside the logo.
