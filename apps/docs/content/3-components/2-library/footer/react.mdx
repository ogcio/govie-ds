# Footer

The Footer component is used to render the bottom section of a web page. This component adheres to accessibility best practices with appropriate roles and aria labels.

The design system now provides two <Code>Footer</Code> components from different import paths:

- **Composable Footer** (recommended): import from <Code>@ogcio/design-system-react/next</Code> — a semantic <Code>{'<footer>'}</Code> wrapper used with <Code>FooterSection</Code> and <Code>FooterLogo</Code>.
- **Legacy Footer** (deprecated): import from <Code>@ogcio/design-system-react</Code> — the slot-based API with <Code>primarySlot</Code>, <Code>secondarySlot</Code>, <Code>utilitySlot</Code>, and <Code>logo</Code> props.

For modern frameworks, a new composable Footer architecture has been introduced for better flexibility and support, replacing the slot usage from the legacy Footer. You can now construct a footer like below from three components: <Code>{'<Footer/>'}</Code>, <Code>{'<FooterSection/>'}</Code>, and <Code>{'<FooterLogo/>'}</Code>. For migrating from the legacy footer, we've provided a guide below.

## Migration guide

With the legacy footer component, only one export was provided; <Code>{'<Footer>'}</Code> — with its content props

- primarySlot
- secondarySlot
- utilitySlot

These would directly map to the positions shown below, with the logo auto-placed in the secondary slot.

<Box className="gi-w-full m-0 gi-not-prose border border-black text-black">
  <Box
    id="primary"
    className="h-16 gi-bg-color-surface-system-neutral-layer1 border text-center"
  >
    <Text>Primary</Text>
  </Box>
  <Box
    id="secondary"
    className="flex h-16 gi-bg-color-surface-system-neutral-layer1 border text-center relative"
  >
    <Text className="grow">Secondary</Text>
    <LogoGoldGreen className="absolute right-1" />
  </Box>
  <Box
    id="utility"
    className="h-10 gi-bg-color-surface-system-neutral-layer2 border text-center"
  >
    <Text>Utility</Text>
  </Box>
</Box>

Now the new API allows the developer to fully control the flow. We provide 3 new components in order to achieve the same effect.

- Footer
- FooterSection
- FooterLogo

### Footer

This component is a pure <Code>{'<footer>'}</Code> component for semantic purposes, and accepts styling, and aria attributes.

### FooterSection

This component is a <Code>{'<Box/>'}</Code>, and comes in two variants

- primary (default)
- utility

Both primary and secondary slots can be placed together within one FooterSection, with a divider between the each. Alternatively, for semantic purposes, you may use two FooterSections representing each - the choice is yours.

The utility slot can also be generated with this component, using the variant prop <Code>utility</Code>.

### FooterLogo

This is a wrapper component for your logo, that self-positions itself to the right of its row for screen-widths greater than <Code>md</Code>. For mobile screens it aligns itself to the left of the footer.

#### Migrating the <Code>logo</Code> prop

The legacy <Code>logo</Code> prop accepted <Code>href</Code>, <Code>alt</Code>, <Code>imageLarge</Code>, <Code>imageSmall</Code>, <Code>width</Code>, and <Code>height</Code>, and automatically rendered a linked logo in the secondary area. In the composable API, you place the logo inside <Code>FooterLogo</Code> and control the image yourself, placing a link within if needed:

```jsx
// Before (legacy)
<Footer
  logo={{
    href: '/',
    alt: 'Gov.ie Logo',
    imageLarge: '/logo-desktop.svg',
    imageSmall: '/logo-mobile.svg',
  }}
  secondarySlot={...}
/>

// After (composable)

<FooterSection>
  <Container>
  ...
    <Stack direction={{ base: 'column', md: 'row' }} gap={4} wrap>
      {/* secondarySlot content */}
      <FooterLogo>
          <LogoGoldGreen size="181" label="Gov.ie Logo" />
      </FooterLogo>
    </Stack>
  </Container>
</FooterSection>
```

If you used custom <Code>imageLarge</Code> / <Code>imageSmall</Code> sources, pass in a custom <Code>size</Code> prop to <Code>LogoGoldGreen</Code> (or any other logo). Alternatively, use your own <Code>{'<img>'}</Code> or <Code>{'<picture>'}</Code> element. <Code>FooterLogo</Code> handles positioning only — it does not add a link or alt text for you.

For migrating over to the new composable footer API, we would first recommend porting over your <Code>primarySlot</Code>, <Code>secondarySlot</Code> and <Code>utilitySlot</Code> directly in the children of the FooterSections like below.

```jsx
import {
  Footer,
  FooterSection,
  FooterLogo,
} from '@ogcio/design-system-react/next';
...
const App = () => {
  return (
    <Footer>
      <FooterSection>
        <Container>
          Primary: used for navigational links, summaries, or descriptive blocks
        <Divider className="gi-my-8" />
        <Stack direction={{ base: 'column', md: 'row' }} gap={4} wrap>
          Secondary: Optional secondary content like metadata or partner links
          <FooterLogo>
            <LogoGoldGreen/>
          </FooterLogo>
        </Stack>
        </Container>
      </FooterSection>
      <FooterSection variant="utility">
        <Container>
          Utility: Typically used for legal links such as privacy policy, accessibility statement
        </Container>
      </FooterSection>
    </Footer>
  )
}
```

The <Code>{'<Container>'}</Code> is a useful way of keeping horizontal padding consistent and responsive across your site. See the [Container docs](/components/library/container/design/) for more information.

A <Code>Stack</Code> with conditional directions based on screen width is effective for controlling the flow of content, particularly for managing the positioning of the secondary content with the logo. Here we use <Code>{'direction={{ base: \'column\', md: \'row\' }}'}</Code>, which is to say, for any screen widths greater than <Code>md</Code>, use <Code>flex-row</Code>. Any less, and use <Code>flex-col</Code>. This is the recommended flow of content between the secondary links, and the logo.

## i18n Keys

The Footer component supports localisation through i18n keys in React, [find i18n setup here](/components/setup-guides/react/#internationalisation-i18n-guidelines). Below is the list of keys and their default text:

```json
{
  "footer.goToHomePage": "Go to the home page",
  "footer.footer": "Footer",
  "footer.primarySlot": "Primary footer slot",
  "footer.secondarySlot": "Secondary footer slot",
  "footer.utilityLinks": "Utility links"
}
```

These keys represent visible text elements. You can customise their translations for each locale to enable multilingual support.
