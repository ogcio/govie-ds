# Footer

The Footer component is used to render the bottom section of a web page. This component adheres to accessibility best practices with appropriate roles and aria labels.

For modern frameworks such as React and Angular, a new composable Footer has been developed to enable higher level flexibility and support, replacing the slot usage from the legacy Footer. The migration guide can be found below.

# Migration from Legacy Footer to Composable Footer

With the legacy footer component, only one export was provided; `<Footer>` - with its content props

- primarySlot
- secondarySlot
- utilitySlot

These would directly map to the positions shown below, with the logo auto-placed in the secondary slot.

<Box className="gi-w-full m-0 gi-not-prose border border-black text-black">
  <Box
    id="primary"
    className="h-16 gi-bg-color-surface-system-neutral-layer1 border text-center"
  >
    <Text>Primary</Text>
  </Box>
  <Box
    id="secondary"
    className="flex h-16 gi-bg-color-surface-system-neutral-layer1 border text-center relative"
  >
    <Text className="grow">Secondary</Text>
    <LogoGoldGreen className="absolute right-1" />
  </Box>
  <Box
    id="utility"
    className="h-10 gi-bg-color-surface-system-neutral-layer2 border text-center"
  >
    <Text>Utility</Text>
  </Box>
</Box>

Now the new API allows the developer to fully control the flow. We provide 3 new components in order to achieve the same effect.

- Footer
- FooterSection
- FooterLogo

### Footer

This component is a pure `<footer>` component for semantic purposes, and accepts styling, and aria attributes.

### FooterSection

This component is a `<Box>` component, and comes in two variants

- primary (default)
- utility

Both primary and secondary slots can be placed together within one FooterSection, with a divider between the each. The utility slot can also be generated with this component, using the variant prop `utility`.

### FooterLogo

This is a wrapper component for the Logo, that self-positions itself to the right of its row for screen-widths greater than `md`.

For migrating over to the new composable footer API, we would first recommend porting over your `primarySlot`, `secondarySlot` and `utilitySlot` directly in the children of the FooterSections like below.

```jsx
import {
  Footer,
  FooterSection,
  FooterLogo,
} from '@ogcio/design-system-react/next';
...
const App = () => {
  return (
    <FooterNext className="gi-not-prose">
      <FooterSection>
        <Container>
          Primary: used for navigational links, summaries, or descriptive blocks
        <Divider className="gi-my-8" />
        <Stack direction={{ base: 'column', md: 'row' }} gap={4} wrap>
          Secondary: Optional secondary content like metadata or partner links
          <FooterLogo>
            <LogoGoldGreen/>
          </FooterLogo>
        </Stack>
        </Container>
      </FooterSection>
      <FooterSection variant="utility">
        <Container>
          Utility: Typically used for legal links such as privacy policy, accessibility statement
        </Container>
      </FooterSection>
    </FooterNext>
  )
}
```

The `<Container>` is a useful way of keeping horizontal padding consistent and responsive across your site. See the [Container docs](/components/library/container/design/) for more information.

A `Stack` with conditional directions based on screen width is highly effective at controlling the flow of content, particularly for managing the positioning of the secondary content with the logo. Here, using `column` for mobile screen widths is recommended.

For a complete example, as show at the beginning of this page, click "Show Code" in the [Footers story](/storybook-react/docs/layout-footer-footer--docs&globals=viewport.isRotated:!false).

## i18n Keys

The Footer component supports localisation through i18n keys in React, [find i18n setup here](/components/setup-guides/react/#internationalisation-i18n-guidelines). Below is the list of keys and their default text:

```json
{
  "footer.goToHomePage": "Go to the home page",
  "footer.footer": "Footer",
  "footer.primarySlot": "Primary footer slot",
  "footer.secondarySlot": "Secondary footer slot",
  "footer.utilityLinks": "Utility links"
}
```

These keys represent visible text elements. You can customise their translations for each locale to enable multilingual support.
