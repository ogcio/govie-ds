## Usage

Icon components should be used to visually support actions or statuses. They work best alongside text in buttons, inputs, or navigation elements, helping users quickly recognise meaning.

### Do

- Pair icons with clear text labels for primary or complex actions.
- Provide accessible names (for example, `aria-label`) when an icon appears without visible text.

### Don't

- Don't use ambiguous symbols users may not recognise.
- Don't rely on icon-only buttons for primary actions.
- Don't omit focus styles or accessible names for assistive technologies.
