# Summary List

This component provides a way to present structured information in rows of labels and values. It is commonly used for displaying a user’s details before confirmation or submission.

## Usage

Use the summary list component to display structured information in a way
that is easy to scan. Each piece of data is shown as a row with a label,
value, and optional action.

### Do

- Use for **reviewable information** (e.g., pre-submit summary).
- Keep labels **short and descriptive**.
- Include actions only when users can take action on the item.
- Use **line breaks** for multi-line content (e.g., addresses).
- Clamp **labels** when space is fixed (optional): `nowrap + ellipsis` with a sensible `max-width`.

<SummaryList withBorder>
  <SummaryListHeader
    overflow="truncate"
    maxWidth="250px"
    label="This is a very, very long summary card heading that demonstrates clamping/ellipsis behavior in the header area so it doesn’t overflow columns"
>
    <SummaryListAction href="/edit-all">Edit all</SummaryListAction>
  </SummaryListHeader>

<SummaryListRow withBorder label="Full name">
  <SummaryListValue>John Smith</SummaryListValue>
  <SummaryListAction href="/edit/name">Change</SummaryListAction>
</SummaryListRow>

<SummaryListRow withBorder label="Date of birth">
  <SummaryListValue>8 November 1982</SummaryListValue>
  <SummaryListAction href="/edit/dob">Change</SummaryListAction>
</SummaryListRow>

<SummaryListRow withBorder label="Address">
  <SummaryListValue>72 Guild Street - London - SE23 6FH</SummaryListValue>
  <SummaryListAction href="/edit/address">Change</SummaryListAction>
</SummaryListRow>

  <SummaryListRow label="Contact details">
    <SummaryListValue>
      07700 864523 - john.smith@example.com
    </SummaryListValue>
    <SummaryListAction href="/edit/contact">Change</SummaryListAction>
  </SummaryListRow>
</SummaryList>

### Don't

- Don't overload a row; **split** large content into multiple rows.
- Don't include **irrelevant actions** or more than a few actions per row.
- Don't truncate **critical data** (IDs, totals); avoid clamping where meaning may change.
