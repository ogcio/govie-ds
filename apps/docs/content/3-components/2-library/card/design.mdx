# Card

The Card component is a versatile container used to group related content and actions. It supports multiple layouts (horizontal, vertical), media content, and contextual metadata like tags, headings, and actions.

## Usage

Cards are ideal for displaying grouped content that needs to be visually separated, such as product information, articles, or user profiles.

### Do

- **Use appropriate layout**: Choose horizontal or vertical type based on your content.
- **Use meaningful media**: Add images, icons, or iframes to enhance context.
- **Provide a clear hierarchy**: Titles, subtitles, and descriptions should be distinguishable.

**Example:**

<ComponentContainer>
  <Card type="vertical">
    <CardMedia
      media={{
        type: 'image',
        config: {
          src: 'https://placeholderjs.com/400x300',
        },
      }}
    />
    <CardContainer>
      <CardHeader>
        <CardTitle>Card Title</CardTitle>
        <CardSubtitle>Subheading</CardSubtitle>
      </CardHeader>
      <CardDescription>This is Card.</CardDescription>
      <CardAction>
        <Button variant="secondary">Button</Button>
      </CardAction>
    </CardContainer>
  </Card>
</ComponentContainer>

### Don't

- **Don’t overuse media**: Use visuals only when they enhance the card’s purpose.
- **Don’t nest unrelated content**: Keep the card content focused.
- **Don’t ignore accessibility**: Use appropriate labels and alt text for media.

**Example:**

<ComponentContainer>
  <Card>
    <CardContainer>
      <CardHeader>
        <CardTitle>Card Title</CardTitle>
      </CardHeader>
      <CardDescription>
        Card with no media or action – not ideal for interactive UIs.
      </CardDescription>
    </CardContainer>
  </Card>
</ComponentContainer>
