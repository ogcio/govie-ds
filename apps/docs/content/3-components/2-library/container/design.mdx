# Container

Use the Container to centre page content, keep a comfortable line length, and line sections up with the rest of the layout. You can add side padding and extra vertical space when a section needs it.

## Layout

The Container is one wrapper around your content. By design, it responsively changes it discretely, respective of screen size and centres the block. See the **Max widths** section in the React tab for a visual example.

### Container props

- **`gutters`** — On by default. Side padding so text and controls do not sit on the very edge of the container. Set to **`false`** if you need the same width but with content that goes edge to edge.
- **`inset`** — Adds space above and below the block. The spacing increases on larger screens.
- **`maxWidth`** — Use a narrower cap, or full width, when a section should not use the default container width.

You can pass **`className`**, and use **`id`** or **`dataTestId`** for styling hooks or tests.

### Container classes (HTML only)

- `gi-layout-container`: Default container with Tailwind’s container behaviour from the `md` breakpoint.
- `gi-layout-container-full-width`: Full-width layout capped at 2xl (1536px).
- `gi-layout-container-inset`: Container with inset padding from the `md` breakpoint.
- `gi-layout-container-max-xl`: Max-width capped at 1280px for optimal readability.

## Usage

Use the Container for page regions and sections that should share the same alignment and width as the rest of the service. It is a simple building block, not a full page layout on its own.

### Do

- Use it to wrap main content and sections that should read and align consistently.
- Turn on **`inset`** when a block needs clear separation from what sits above and below.
- Rely on **`gutters`** and **`maxWidth`** when you need edge-to-edge content or a different cap without rolling custom width rules each time.
- Set **`dataTestId`** when automated tests need a reliable handle on the wrapper.

### Don't

- Rely on it for complex layout (multi-column pages, nested grids, or heavy flex). Reach for layout utilities or a more specialised pattern instead.
- Add one inside another unless you have a clear reason; multiple nested wrappers are usually harder to read and reason about.
- Use it when your design does not need a shared width and alignment—plain markup or a lighter wrapper may be clearer.
