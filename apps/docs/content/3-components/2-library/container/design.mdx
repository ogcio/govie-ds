# Container

Use the Container to center page content and keep readable line lengths across breakpoints. It enforces consistent horizontal alignment and optional vertical insets for sections.

## Usage

Use the Container when you need a centralized, consistent wrapper for page sections, ensuring content remains readable and aligned.

### Do

- Use it to provide a centralized, consistent layout wrapper for page content.
- Keep content readable with a capped max-width via **gi-container**.
- Center sections horizontally using **gi-mx-auto**.
- Wrap main content areas or any section that should align to the center for a uniform look and feel.
- Target it in automated tests with **data-testid="govie-container"** to reliably select wrapped content.

### Don't

- Avoid using the container component if you need a flexible or dynamic layout that requires complex positioning, such as grids or flexbox-based structures.
- This component is not suitable for scenarios where you need to override the maximum width or margins frequently, or when the content needs to be fluid across the full width of the viewport.
- If your content does not require centralised alignment or if the predefined styles conflict with the specific requirements of your design, consider using a custom container or a different layout component that offers greater flexibility.
