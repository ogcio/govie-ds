# Progress Stepper

The Progress Stepper component visually represents a user's position in a multi-step process such as a form, onboarding flow, or application submission.

## Variants

There are two main orientations for the Progress Stepper: horizontal and vertical.
The horizontal orientation is the default and is suitable for most use cases, while the vertical orientation is useful when steps include forms or require more detailed content.

### Horizontal (Default)

## Usage

Use the Progress Stepper to visually divide a process into steps and indicate progress through them. It's especially useful for guiding users through forms or tasks that must be completed in order.

### Do

- Use clear and descriptive labels for each step.

- Use vertical orientation when steps include form content or require user input.
- Keep steps concise and meaningful.
- Each step should represent a distinct part of the process. Avoid breaking steps into overly granular sub-steps unless necessary.

<ComponentContainer>
  <ProgressStepper currentStepIndex={1}>
    <StepItem label="Start Application" />
    <StepItem label="Personal Info" />
    <StepItem label="Document Upload" />
    <StepItem label="Review & Submit" />
  </ProgressStepper>
</ComponentContainer>

### Don't

- Don’t use this for non-linear flows where steps can be completed in any order.
- Don’t overload steps with too much content—split into substeps or screens if needed.
- Avoid showing more steps than necessary; reduce cognitive load for users.

<ComponentContainer>
  <ProgressStepper currentStepIndex={0}>
    <StepItem label="Step A (optional)" />
    <StepItem label="Step B (optional)" />
    <StepItem label="Step C (optional)" />
  </ProgressStepper>
</ComponentContainer>
