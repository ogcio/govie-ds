A new version of the `Select` component is now available: **`SelectNext`**.

We recommend using `SelectNext` whenever native HTML elements `<select>` and `<option>` are not strictly required. The new version offers enhanced user experience, improved accessibility, and greater visual and functional control.

The native `Select` component is **currently deprecated** and will be **fully replaced by `SelectNext`** in a future release.

### When to continue using `Select`

- When rendering a native HTML `<select>` element is strictly necessary

For further information, please refer to the `SelectNext` storybook.
