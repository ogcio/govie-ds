# Select

The Select component allows users to choose a single option from a predefined list.

## Usage

The select component should only be used as a last resort in public-facing services because research shows that some users find selects very difficult to use.

### Do

- **Limit the number of options**: Use a select when there are more than 5-8 options but fewer than 20-30. For fewer options, consider radio buttons. For many more, consider an auto-completing input or search.
- **Provide a clear default or "empty" state**: Include a default option like "Select an option..." or "None" to indicate that a choice needs to be made.
- **Order options logically**: Arrange options alphabetically, numerically, or by common usage to make them easy to find.
- **Use clear and concise labels**: Ensure each option's text is unambiguous and easily understood.
- **Consider accessibility**: Ensure the select component is keyboard navigable and correctly conveys its state and options to assistive technologies.

<ComponentContainer>
  <div className="flex flex-col gap-4">
    <FormField>
      <FormFieldLabel>Country</FormFieldLabel>
      <Select aria-label="Country" defaultValue="">
        <SelectItem value="" hidden>
          {'Select a country'}
        </SelectItem>
        <SelectItem value="us">{'United States'}</SelectItem>
        <SelectItem value="ca">Canada</SelectItem>
        <SelectItem value="mx">Mexico</SelectItem>
      </Select>
    </FormField>
    <FormField>
      <FormFieldLabel>Favorite Color</FormFieldLabel>
      <Select aria-label="Favorite Color" defaultValue="blue">
        <SelectItem value="red">Red</SelectItem>
        <SelectItem value="blue">Blue</SelectItem>
        <SelectItem value="green">Green</SelectItem>
      </Select>
    </FormField>
  </div>
</ComponentContainer>

### Don't

- **Don't use for true/false or binary choices**: For two options, use checkboxes or toggle switches instead.
- **Don't hide critical information**: If all options are equally important and need to be visible at a glance, a select is not appropriate.
- **Don't use for actions or commands**: Selects are for choosing data, not for triggering actions. Use buttons or links for actions.
- **Don't put too many options in one select**: If the list becomes excessively long (e.g., hundreds of items), consider a different UI pattern like a search-enabled dropdown or a dedicated search page.
- **Don't rely solely on placeholder text for critical choices**: Always use a clear label for the select component itself.
