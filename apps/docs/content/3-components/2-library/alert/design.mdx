# Alert

The Alert component is used to display important feedback messages to users. It supports different variants like success, warning, info, and danger, and may be dismissible.

## Usage

Use alerts to provide contextual feedback messages for typical user actions. These messages are often used to inform users of success, error, or warning states.

### Do

- Use appropriate variants for different message contexts (info, success, warning, danger).
- Keep messages short and actionable.
- Use dismissible alerts when users can safely ignore the message.
- Place alerts in prominent locations within the UI.
- Provide accessible labels and roles.

**Example:**

<ComponentContainer>
  <Alert title="Update saved" variant="success" dismissible>
    Your profile information was successfully updated.
  </Alert>
</ComponentContainer>

### Don't

- Don't use alerts to display large blocks of text.
- Don't use alerts for interactive content.
- Don't show multiple alerts stacked unnecessarily.
- Don't rely solely on color to convey meaning.
- Don't use alerts for permanent messages.

**Example:**

<ComponentContainer>
  <Alert title="This alert contains too much content and interactivity">
    <div>
      <button>Click Me</button>
      <p>This alert is trying to do too much and overwhelms the user.</p>
    </div>
  </Alert>
</ComponentContainer>

## i18n Keys

The Alert component supports localisation through i18n keys in React, [find i18n setup here](/components/setup-guides/react/#internationalisation-i18n-guidelines). Below is the list of keys and their default text:

```json
{
  "alert.dismissAlert": "Dismiss Alert"
}
```

These keys represent visible text elements. You can customise their translations for each locale to enable multilingual support.
