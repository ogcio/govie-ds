# Score Select

Score Select is used to collect a rating score from users, typically in the form of Net Promoter Score (NPS). It displays a horizontal row of buttons representing numeric values from 0 to 10, and includes labels on either side to help frame the scale.

## Usage

### ScoreSelect Props

- `label` (required): The main label displayed above the button group.
- `hint` (optional): A brief description or instruction shown under the label.
- `name` (required): The name attribute used for form submission and identification of the group of buttons.
- `type` (required): Defines the scale of the button group. It can be one of the following:
  - `'1-5'`: A scale from 1 to 5.
  - `'1-7'`: A scale from 1 to 7.
  - `'0-10'`: A scale from 0 to 10.
- `leftLabel` (optional): The text that appears on the left side of the score range (e.g., "Not Likely").
- `rightLabel` (optional): The text that appears on the right side of the score range (e.g., "Extremely Likely").
- `size` (optional): The size of the buttons. Options include:
  - `'small'`
  - `'medium'`
  - `'large'`
    The default size is `medium`.
- `onChange` (optional): Callback function when the selected score changes.

### Best Practices

#### When to use this component

Use the `ScoreSelect` component when you need a quick and intuitive way to collect user ratings, such as:

- **Net Promoter Score (NPS)** surveys.
- **Satisfaction ratings** or **feedback forms** requiring numeric ratings.
- Collecting user **opinions** on a Likert scale.

#### When not to use this component

Avoid using this component if:

- You need **text-based answers** or **qualitative feedback**.
- You require more than **one score group** in a single input.
- A **slider** or **range input** is more appropriate for continuous values.
