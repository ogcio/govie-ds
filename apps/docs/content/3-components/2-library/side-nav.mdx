# Side Navigation

Side Navigation (SideNav) helps users navigate across major sections of an application. It is commonly used in dashboard layouts, allowing for quick and persistent access to key areas.

## Usage

### SideNav Props

- **value** (optional): The currently selected item's value. Helps manage active state.
  - Example: `'projects-active'`

### SideNavItem Props

- **value** (required): Unique identifier for the item.
- **label** (required): The visible label text for the item.
- **icon** (optional): Icon name (material symbols) to show beside the label.
- **href** (optional): URL if the item should behave like a link.
- **parent** (optional): Indicates that the item has children.
- **expandable** (optional): Allows the item to be expanded/collapsed.
- **open** (optional): Sets the item to be initially open if expandable.

### Best Practices

#### When to use this component

- When your application requires persistent, structured navigation.
- To allow users to quickly access main sections or dashboards.
- Ideal for admin panels, complex tools, or multi-section content.

#### When not to use this component

- For simple single-page navigation.
- On mobile devices where a drawer or tab-based system might be more appropriate.
- When the visual hierarchy isnâ€™t deep enough to justify a full side nav.
