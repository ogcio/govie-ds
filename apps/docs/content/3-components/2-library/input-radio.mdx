## Usage

<ComponentPropertiesTable componentId="input-radio" />

### Radio Props

- **label** (optional): The label displayed next to the radio input.

- **hint** (optional): An optional hint text to display below the radio input.

- **size** (optional): The size of the radio button. Possible values are:

  - `'lg'` for large
  - `'md'` for medium
  - `'sm'` for small

- **conditionalInput** (optional): Allows for a conditional text input that will be displayed depending on the state of the radio button.

- **children** (optional): Standard HTML attributes for an input element (excluding `size`).

### Radio Group Props

- **groupId** (required): A unique identifier for the radio group.

- **inline** (optional): If true, displays the radio buttons inline.

- **onChange** (optional): A callback function that is triggered when a radio button is selected.

### Best Practices

### When to use this component

Use the radios component when users can only select one option from a list.

### When not to use this component

Do not use the radios component if users might need to select more than one option. In this case, you should use the checkboxes component instead.
