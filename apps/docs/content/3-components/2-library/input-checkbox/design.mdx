# Input Checkbox

The Input Checkbox component allows users to make binary choices such as selecting options, agreeing to terms, or toggling features. It supports multiple sizes, states, and accessible labeling.

## Usage

Use Input Checkbox components when you need to present one or more selectable options, where each option is independent of the others.

### Do

- **Use clear labels**: Ensure every checkbox has a visible label that describes what it does.
- **Group related checkboxes**: When part of a set, present checkboxes together with a group label.
- **Provide hints when needed**: Use hints to guide user decisions where necessary.
- **Use indeterminate state for partial selections**: Show the indeterminate state when only some items in a group are selected, helping users understand that the group is partially selected.

**Example:**

<ComponentContainer>
  <div className="flex flex-col gap-4">
    <InputCheckbox
      id="terms-checkbox"
      label="I agree to the terms and conditions"
    />
    <div className="gi-text-sm gi-text-gray-600">
      Clear label, commonly used for agreement checkboxes.
    </div>
  </div>
</ComponentContainer>

### Don't

- **Don’t leave labels out**: Users relying on screen readers require labels to understand checkbox meaning.
- **Don’t use checkboxes for single mandatory selections**: Use radio buttons or switches when only one choice is valid.
- **Don’t make checkboxes too small to interact with**: Respect size guidelines for accessibility.

**Example:**

  <div className="flex flex-col gap-4">

    </div>
  </div>
</ComponentContainer>
