# Combo Box

The **Combo Box** is a flexible UI pattern that combines an input field with a dropdown list of options.  
It allows users to either select a predefined option or enter a custom value, making it suitable for both controlled and free-form inputs.  
These guidelines describe the recommended visual design, variants, and usage principles to ensure consistency across products.

## Variants

The Combo Box supports several visual and behavioural variants to suit different contexts:

- **Multi-select:** Allows users to select multiple options, often displayed as chips/tags.
- **Searchable:** Enables users to filter options as they type.
- **Disabled:** A non-interactive variant for read-only states.

## Usage

Follow these best practices when using the Combo Box component.

### Do

- Use a Combo Box when the list of options is long, and typing can help users find the right one faster.
- Include clear labels to indicate what is being selected.
- Order options logically, such as alphabetically or by relevance.
- Use placeholder text to guide users when no selection has been made.

<ComponentContainer>
  <Form>
    <Combobox className="gi-mx-auto">
      <DropdownItem
        options={[
          { label: 'Option A', value: 'a' },
          { label: 'Option B', value: 'b' },
          { label: 'Option C', value: 'c' },
        ]}
        defaultValue={['b']}
      >
        {'Select an option'}
      </DropdownItem>
    </Combobox>
  </Form>
</ComponentContainer>

### Don't

- Don’t use a Combo Box when there are very few options; use radio buttons or a select menu instead.
- Don’t pre-fill the input with a misleading or irrelevant default value.
- Don’t disable typing if user-provided values are expected.
- Don’t overload the component with too many complex options.

  <Form>

        defaultValue={['only']}

      >
        {'Avoid this for single options'}
      </DropdownItem>
    </Combobox>
  </Form>
</ComponentContainer>
