# Button Group

A button group, also known as a segmented control, is a set of buttons grouped together to allow users to select one option from a set.
It is commonly used for toggling between views, filtering content, or selecting mutually exclusive options.
Button groups help streamline the user interface by consolidating related actions into a single component.

## Usage

### Dos

- **Use button groups for mutually exclusive options**: Button groups are ideal for scenarios where users need to select one option from a set, such as toggling between views or filtering content.
- **Keep labels concise**: Ensure button labels are short and descriptive to avoid clutter and improve readability.
- **Maintain visual consistency**: Use consistent spacing, alignment, and styling to ensure the button group integrates seamlessly with the rest of the UI.
- **Provide clear feedback**: Highlight the selected button to indicate the current state and provide hover and focus states for better accessibility.

**Example:**

<ComponentContainer>
  <ButtonGroup name={'opinion_scale'} size={'large'} defaultValue={2}>
    <ButtonGroupItem key={1} value={1}>
      List View
    </ButtonGroupItem>
    <ButtonGroupItem key={2} value={2}>
      Grid View
    </ButtonGroupItem>
  </ButtonGroup>
</ComponentContainer>

### Don'ts

- **Avoid using too many buttons**: Limit the number of buttons in a group to avoid overwhelming users. If there are too many options, consider using a dropdown or another component.
- **Don’t use button groups for unrelated actions**: Ensure all buttons in the group are related and part of the same decision-making process.
- **Avoid ambiguous labels**: Ensure button labels are clear and unambiguous to prevent user confusion.
- **Don’t ignore accessibility**: Avoid designs that rely solely on color to indicate state. Use additional indicators like borders or icons for better accessibility.
