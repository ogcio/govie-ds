# Button

Buttons allow users to take actions, make choices, and navigate interfaces with a single tap.
This versatile component comes in multiple variants, sizes, and styles to accommodate different design needs and user interactions throughout your application.

- **Primary**: The main action button that stands out and draws attention. Use it for the most important actions on a page.
- **Secondary**: A less prominent button that complements the primary button. Use it for secondary actions that are still important but not the main focus.
- **Flat**: A button with minimal styling, often used for less important actions or in contexts where a more subtle appearance is desired.

## Usage

Buttons are interactive elements that enable users to trigger actions or navigate through an application. Using buttons appropriately creates a more intuitive and efficient user experience.

### Do

- **Use clear, action-oriented labels**: Button text should be concise and describe the action that will occur (e.g., "Save changes", "Submit form", "Continue").
- **Maintain hierarchy**: Use primary buttons for main actions, secondary buttons for alternative actions, and flat buttons for less important actions.
- **Ensure adequate spacing**: Allow enough space between multiple buttons to prevent accidental clicks.
- **Provide visual feedback**: Ensure buttons have hover, active, and focus states to indicate interactivity.
- **Be consistent**: Use the same button style for similar actions across your application.
- **Include icons when helpful**: Add icons to buttons when they enhance understanding, but ensure they're relevant to the action.

**Example:**

    </div>

  </div>

### Don't

- **Don't use buttons for navigation**: Use links instead of buttons when the purpose is to navigate to a different page.
- **Don't use multiple primary buttons**: Avoid having more than one primary button in a section as it can confuse users about the main action.
- **Don't use vague labels**: Avoid generic labels like "Click here" or "Submit" without context.
- **Don't disable buttons without explanation**: If a button is disabled, provide clear information about why and how to enable it.
- **Don't overcrowd interfaces with buttons**: Limit the number of buttons in one view to reduce cognitive load.
- **Don't use button groups for unrelated actions**: Button groups should only contain buttons that are related to each other.
