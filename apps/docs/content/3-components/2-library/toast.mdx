## Usage

<ComponentPropertiesTable componentId="toast" />

### Toast Props

- **variant** (optional): Specify the variant of the toast component.
  - Options: `['info', 'danger', 'success', 'warning']`
- **animation** (optional): Specify the toast animation.
  - Options: `['fadeinup', 'fadeinleft', 'fadeinright']`
- **title** (optional): Specify the title of the toast component.
- **dismissible** (optional): Specify if the toast is dismissible.
- **description** (optional): Specify the content in the toast component.
- **action** (optional): Specify a link for the toast component.
- **slotAction** (optional): Specify a custom action for the toast component.
- **duration** (optional): Set the duration of the toast appearing on screen.
- **position** (optional): Specify the position of the toast.
  - Type: `object`
  - Example: `x: ['left', 'center', 'right'] y: ['top', 'center', 'bottom']`

### Best Practices

- Use dismissible toasts when the message is not critical.
- Keep animations smooth and consistent with UI interactions.
- Positioning matters: Important alerts should appear in prominent areas like top-right or center.

### When to use this component

- System notifications (e.g., "Form submitted successfully.")
- Warnings & Errors that require user attention.
- Temporary alerts that disappear automatically.

### When not to use this component

- Permanent messages → Use modals or banners instead.
- Multiple simultaneous alerts → Can overwhelm users.
